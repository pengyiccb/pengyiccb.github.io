    1. worker抢占机制

    主进程master根据配置创建三个worker进程，当客户端请求过来的时候会有一个互斥锁accept_mutex，worker进程就去争抢这个锁。

    当worker1抢到了之后就去处理客户端分发过来的请求和响应。Nginx是采用异步非阻塞通信模式，所以当有客户端请求到worker，状态

    是阻塞的话，worker会处理后续的请求。worker默认配置的客户端最大连接数是1024，通过events模块下的worker_connections配置。

    1. Nginx常用命令

    ./nginx -s stop 强制关闭。可以针对非法请求

    ./nginx -s quit 如果用户有请求的话，会等待用户请求完毕再关闭

    ./nginx -t 检查配置文件语法是否正确

    ./nginx -v 查看Nginx的版本号。大v的话则会显示更多的信息

    ./nginx -h 显示帮助信息

    ./nginx -c filename 指定配置文件

    1. 手动Nginx日志切割

    进入Nginx的可执行文件目录，创建一个shell可执行文件cut_my_log.sh，内容为：

    #!/bin/bash
    LOG_PATH = “/var/log/nginx”
    RECODE_TIME = $(date -d “yesterday”+%y-%m-%d+%H:%M)
    PID = /var/run/nginx/nginx.pid
    mv ${LOG_PATH }/access.log ${LOG_PATH }/access.${RECODE_TIME }.log
    #向主进程发送信号，用于重新打开日志文件
    kill -USR1 ‘cat $PID’
    为cut_my_log.sh添加可执行权限

    chmod +x cut_my_log.sh

    1. 定时任务日志切割

    1) 安装定时任务

    yum install crontabs
    2) crontab -e 添加并编辑一行新的任务。

    */1 * * * * $NGINX_HOME/sbin/nginx/cut_my_log.sh
    3) 重启定时任务

    service crond restart
    4) 常用的命令

    service crond start #启动服务
    service crond stop #关闭服务
    service crond restart #重启服务
    service crond reload #重新载入配置
    crontab -e # 编辑任务
    crontab -l #查看定时任务列表﻿

    1. gzip压缩功能，提升传输功能，节约带宽

    gzip on #开启gzip压缩功能

    gzip_min_length 1 #限制最小压缩，小于1字节不压缩

    gzip_conp_level 3 #定义压缩的级别（压缩比，文件越大，压缩越多，但是CPU用的也越多）

    gzip_type text/plain application/javascript #定义压缩文件的类型

    1. location匹配规则

    location / {
    root /home
    } #这种匹配模式能够匹配到/home文件下的
    location = /imooc/img/face1.img {
    root /home
    }# 精准匹配，只有响应了/imooc/img/face1.img才有用}
    location (jpg|png|img) {
    root /home
    }#正则表达式匹配。    区分大小写。
    location ^    /imooc/img {
    root /home
    }# 以某个路径字符开头请求
